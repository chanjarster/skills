# 任务管理

任务有三种状态：`未开始` → `进行中` → `已完成`

## 创建新任务

1. 在 `4-任务日志/0-未开始/` 下创建目录 `<yyMMdd>-<HHmm>-<任务标题>`
2. 使用模板创建 `task.md`（任务描述）
3. 更新 `4-任务日志/0-索引.md`（参见 [5-索引维护规则.md](5-索引维护规则.md)）
4. 询问用户是否立即开始制定开发计划，若用户确认则进入"创建开发计划"流程

## 创建开发计划

当用户要求执行任务、或新任务创建完成后，按以下步骤制定计划：

1. 理解任务内容
2. 搜索和任务相关的 pin-docs 文档，了解业务背景和已有技术设计；若文档不足以厘清实现细节，则进一步通过代码地图定位并阅读相关代码
3. （可选）若经过文档和代码搜索后仍存在不明确之处，向用户提问以澄清；若已完全理解任务，直接进入下一步
4. 厘清所有不明确之后，创建 `plan.md`（开发计划）
5. 创建开发计划之后，不要立即执行。提示用户审核 `plan.md`，等待用户明确授权（如"开始执行"）后再进入执行阶段

## 执行任务

1. 把任务目录从 `0-未开始` 移到 `1-进行中`，并将 `task.md` frontmatter 中 `status` 更新为 `"进行中"`
2. 创建 `result.md`，frontmatter 中 `completed` 暂填空字符串，正文留空；在执行阶段不需要更新进度，仅在"标记完成"阶段写入最终摘要
3. 更新 `4-任务日志/0-索引.md`（状态切换完成）
4. 按照 `task.md` 和 `plan.md` 执行任务，在 `plan.md` 的步骤列表中，将已完成的步骤改为 `- [x]` 格式标记。
5. 按照 `task.md` 中的验收标准逐条验收；可由 AI 自动化执行，也可与用户进行人工确认。

## 标记完成

1. 在 `result.md` 中写入完成摘要
2. 将任务目录从 `1-进行中` 移到 `2-已完成`，并将 `task.md` frontmatter 中 `status` 更新为 `"已完成"`
3. 更新 `4-任务日志/0-索引.md`
