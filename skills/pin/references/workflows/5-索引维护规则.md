# 索引维护规则（必读）

**0-索引.md 是 Pin 的导航核心。任何对文档的增删改操作，都必须同步更新受影响层级的 0-索引.md。这是一条不可跳过的强制规则。**

以下操作必须触发索引更新：

| 操作 | 索引更新内容 |
|---|---|
| 创建文档 | 在所在层级的 `0-索引.md` 添加新条目（标题、摘要、路径） |
| 更新文档 | 在所在层级的 `0-索引.md` 更新该条目的摘要（若内容含义发生变化） |
| 删除文档 | 在所在层级的 `0-索引.md` 移除该条目 |
| 任务状态流转 | 更新 `4-任务日志/0-索引.md` 中该任务的路径和状态 |
| 创建/删除子目录 | 在父层级的 `0-索引.md` 添加/移除该子目录的条目 |

**自检**：每次完成文档增删改操作后，立即检查：`0-索引.md` 是否已反映本次变更？若否，补上。
